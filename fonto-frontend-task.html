Fonto Frontend Task

Context

Users of our system want to see a list of their properties and add new ones.
In the backend, the following data is stored for each property:

  • id : Integer, a unique identifier automatically assigned when adding a property
  • address : Text, free text entered by the user
  • valuation : Numeric, estimated value of the property entered by the user
  • hidden : Boolean, indicates if property should be shown in the frontend

There is a basic REST API to retrieve properties for a user and add new ones.

Requirements

  • Display the list of properties for your user obtained from the API (see API use below for details) except the ones where hidden == true
  • Format valuation as currency with thousands separator

  • Display the sum of the valuation for the displayed properties

  • Add Button to allow the user to add properties and store in the backend using the corresponding API endpoint.

  • Valuation field should only accept numeric values

API use

  • base URL: https://take-home.hasura.app/api/rest
  • You need to include in every request the header x-hasura-user-id with the value EBx22JCTOKbqrsF
  • Responses will be application/json
  • These are the available endpoints:
